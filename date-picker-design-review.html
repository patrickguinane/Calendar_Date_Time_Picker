Pattern 2 — Single Date & Time (Decoupled)

Two Separate Fields
Changed Date button + time input are independent fields side by side
Payment date
Time
UX: Date and time are decoupled. The date button opens a calendar (date-only, no time inside). The time input is a standard input field — each has its own label. This keeps the calendar dialog simple and allows the time to be optional or pre-filled independently.
Empty State
Both fields with placeholders
Payment date
Time
UX: When both are empty, each shows its own placeholder independently. The date button shows "Select date", the time input shows "HH:MM". They validate independently too — you could require one without the other.

Design Decisions Summary

Decision Approach Why
Trigger Button (not input) Prevents free-text entry errors; consistent click target
Label Above button Matches input field pattern; provides context
Clear Inside dialog, keeps open User can immediately re-select after clearing
Frontend errors Inside dialog Error is actionable within the dialog context
Backend errors Hint text below button Consistent with form input error pattern
Single date + time Two separate fields Keeps calendar simple; time can be optional/independent
Range + time Time inside dialog Minimises clicks; one flow for start/end date+time
